# impm-req-create 技能

## 触发词
impm-req-create

## 何时使用
- project.md生成完成后的下一步
- 需要根据用户提示词整理需求文档时

## 执行步骤

### 1. 读取输入材料
- 用户的提示词
- 提示词中提到的需求文档路径

### 2. 确定版本号
使用impm_doc_version工具（action=next）获取版本号：
- 如果提示词或文档中有版本号，优先使用
- 否则扫描docs/requires/目录取最大版本号，修订号+1
- 如果目录下没有文件，使用默认版本号v0.0.1

### 3. 整理需求文档
- 将提示词和需求文档内容合并。
- 分析业务需求，识别痛点和价值点。
- 需求澄清、需求降噪，
- 需求有不明确的地方，直接询问用户
- 需求收集，分析完成后，按以下格式整理：

```markdown
# 需求文档

**项目名称：** {项目名称}
**版本号：** v{x.x.x}
**日期：** {当前日期}

## 项目背景
{背景描述}

## 功能需求
### FR-001: {需求名称}
- 描述：{详细描述}
- 优先级：{高/中/低}
- 验收标准：{标准}

## 非功能需求
### NFR-001: {需求名称}
- 描述：{详细描述}
- 指标：{可量化的指标}

## 约束条件
{约束描述}

## 假设与依赖
{假设和依赖描述}
```

如有需要，可以通过搜索本地代码（用CS subagent）和网上查询（用WS subagent）收集需求业务相关信息，完善文档。

### 4. 写入文档
使用impm_doc_writer将需求文档写入：
`docs/requires/{项目名称}-requirement-v{x.x.x}.md`
将docs/project.md 的当前进度改为：impm-req

### 5. 返回文档位置
返回生成的文档路径给PM agent。



## 交付物
- docs/requires/{项目名称}-requirement-v{x.x.x}.md

## 注意事项
- 版本号确定后，后续所有文档和程序版本都使用此版本号
- 需求细节要完整，如有不明确之处应向用户提问
- 所有内容使用简体中文
