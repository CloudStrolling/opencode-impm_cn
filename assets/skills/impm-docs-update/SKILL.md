# impm-docs-update 技能

## 触发词
impm-doc-update

## 何时使用
- 所有代码变更任务完成后
- 需要更新项目文档时

## 执行步骤

### 1. 读取项目设置
使用impm_doc_reader读取project.md，确认：
- 本地语言设置
- 项目名称和描述
- 编程语言和技术栈

### 2. 生成README.md
根据项目信息生成完整的README.md，包含：

```markdown
# {项目名称}

{项目简介描述}

## 功能特性
{主要功能列表}

## 安装
\`\`\`bash
# 安装步骤
\`\`\`

## 使用方法
{使用说明}

## 开发指南
\`\`\`bash
# 开发环境搭建
# 运行测试
# 构建项目
\`\`\`

## 项目结构
{目录结构说明}

## 技术栈
{使用的技术列表}

## 许可证
{许可证信息}
```

### 3. 生成agent说明文档
如果有自定义agent，生成agent.md说明文档。

### 4. 生成编译和部署文档
- 编译步骤说明
- 环境配置要求
- 部署步骤和脚本
- 环境变量说明

### 5. 生成部署脚本
根据技术栈生成部署脚本：
- Docker配置（如适用）
- CI/CD配置
- 环境变量模板
- 数据库脚本

### 6. 生成部署脚本
根据技术栈生成部署脚本：
- Docker配置（如适用）
- CI/CD配置
- 环境变量模板

### 7. 完成流程
返回PM agent，所有任务完成。

## 交付物
- README.md
- agent.md（如有自定义agent）
- 编译部署文档
- 部署脚本
- git提交和合并记录

## 注意事项
- 所有文档使用简体中文（根据project.md的本地语言设置）
- README要足够详细，新开发者能根据README搭建环境
- 部署文档要包含所有必要的环境配置
- 合并到主分支前确认所有代码和测试正常
