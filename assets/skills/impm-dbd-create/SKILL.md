# impm-dbd-create 技能

## 触发词

impm-dbd-create

## 何时使用

- task完成之后


## 执行步骤

### 1. 判断是否需要创建数据库文档
1. 如果技能启动时强制要求更新 dbd.md，就跳过第二条判断，直接执行第三条判断：project.md是否存在数据库设置。
2. docs/dbd.md是否存在，如果存在就跳过。
3. 如果不存在，查询一下project.md里是否本项目不需要数据库或者没有定义数据库。如果是这样，跳过。
4. 询问agent自己，当前是否有连接到数据库。如果没有连接到数据库就跳过。
5. 如果有连接到数据库，project.md也明确定义需要数据库，则开始创建 dbd.md。


### 2. 读取数据库的设计文档模型
数据库设计文档模版和数据库设计文档创建技能放在一个目录下:DBD-TEMPLATE.MD
首先读取这个模块，根据project.md里的内容，结合agent自己的专业判断，完成模版中项目数据库选型，命名规范，安全与权限，容灾与备份的编写。

### 3. 读取数据库实际内容
通过数据库MCP读取所有的表，字段，索引，分库分表等信息，完成模版中物理模型与数据字典的编写。

### 4. 写入文档

使用impm_doc_writer将dbd写入：
`docs/dbd.md`
然后将docs/project.md 的当前进度改为：impm-dbd

### 5. 返回文档位置

返回生成的文档路径给PM agent。

## 交付物

- docs/dbd.md

## 注意事项
1. 如果要在项目中使用数据库，请先在项目级里面定义连接数据库的mcp。同时连接的数据库要和project.md中的数据库一致。
2. 这里只是保存当前数据库的架构和物理模型。实际数据库的设计，在编码时设计。否则容易造成编码和数据库设计脱节，后续需要返工。dbd.md在编码时的数据库设计时，还会更新。
