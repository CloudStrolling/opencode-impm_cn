# impm-architect-update 技能

## 触发词
impm-architect-update

## 何时使用
- PRD文档生成完成后
- 判断：如architecture.md不存在，则创建
- 判断：如已存在，但PRD涉及架构变更，则更新
- 判断：如已存在且不涉及架构变更，则跳过

## 执行步骤

### 1. 读取前序文档
使用impm_doc_reader读取：
- PRD文档
- project.md
- 现有的architecture.md（如有）

### 2. 分析架构需求
- 分析PRD中的功能需求对架构的影响
- 确定需要的模块划分
- 确定技术选型
- 确定数据流和交互方式

### 3. 生成/更新架构文档
按以下格式生成architecture.md：

```markdown
# 架构文档

**项目名称：** {项目名称}
**版本号：** v{x.x.x}
**日期：** {当前日期}

## 系统架构概述
{总体架构描述，包含架构图（文字描述）}

## 模块设计
### 模块1: {名称}
- 职责：{描述}
- 依赖：{依赖的其他模块}
- 接口：{对外暴露的接口}

### 模块2: {名称}
...

## 技术选型
| 领域 | 技术方案 | 理由 |
|------|---------|------|
| 前端框架 | ... | ... |
| 后端框架 | ... | ... |
| 数据库 | ... | ... |

## 数据流
{数据在系统中的流转描述}

## 目录结构
{项目目录结构规范}

## 部署架构
{部署方案描述}
```

### 4. 架构确认
如在生成过程中有需要明确的架构要求，向用户提问直至细节清晰。
如需查询相关信息，可使用网络搜索获取技术文档。

### 5. 写入文档
使用impm_doc_writer将架构文档写入docs/architecture.md。

### 6. 返回结果
返回文档路径或"无需更新"给PM agent。

## 交付物
- docs/architecture.md 文件

## 注意事项
- 首次创建需要完整生成
- 更新时保留未变更部分，只修改相关内容
- 架构设计要考虑可扩展性和可维护性
- 所有内容使用简体中文
