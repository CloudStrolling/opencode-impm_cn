# impm-spec-create 技能

## 触发词
impm-spec-create

## 何时使用
- PRD文档生成完成后
- 架构文档生成完成后（如有）
- 需要将PRD和架构转化为具体技术规格时

## 执行步骤

### 1. 读取前序文档
使用impm_doc_reader读取：
- PRD文档
- project.md
- ARCHITECTURE.md（如有）

### 2. 生成spec
根据PRD和架构文档，生成技术规格说明：

```markdown
# 技术规格文档

**项目名称：** {项目名称}
**版本号：** v{x.x.x}
**日期：** {当前日期}

## 技术方案概述
{总体技术方案描述}

## 模块详细设计

### 模块1: {名称}
#### 功能描述
{详细功能说明}

#### 接口定义
```typescript
interface Example {
  // 接口定义
}
```

#### 数据模型
```typescript
interface DataModel {
  // 数据结构定义
}
```

#### 错误处理
| 错误码 | 描述 | 处理方式 |
|--------|------|---------|
| E001 | ... | ... |

## 通用设计

### 错误处理方案
{统一错误处理策略}

### 日志方案
{日志记录策略}

### 安全方案
{安全相关设计}

## 依赖清单
| 包名 | 版本 | 用途 |
|------|------|------|
| ... | ... | ... |
```

### 3. 需求确认
如有不明确的技术细节，向用户提问直至清晰。

### 4. 写入文档
使用impm_doc_writer将spec写入：
`docs/specs/{项目名称}-spec-v{x.x.x}.md`

### 5. 返回文档位置
返回生成的文档路径给PM agent。

## 交付物
- docs/specs/{项目名称}-spec-v{x.x.x}.md

## 注意事项
- spec要足够详细，使编码工程师可以直接根据spec实现
- 接口定义要包含参数类型和返回值
- 错误处理要覆盖所有已知异常场景
- 所有内容使用简体中文
