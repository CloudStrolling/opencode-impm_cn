# impm-prd-create 技能

## 触发词
impm-prd-create

## 何时使用
- 需求文档生成完成后的下一步
- 需要将需求转化为UserStory格式时

## 执行步骤

### 1. 读取前序文档
使用impm_doc_reader读取：
- 前序生成的requirement文档
- project.md

### 2. 生成PRD
根据需求文档和project.md，生成格式完整的PRD文档，采用UserStory方式记录：

```markdown
# PRD文档

**项目名称：** {项目名称}
**版本号：** v{x.x.x}
**日期：** {当前日期}

## 产品概述
{产品整体描述}

## 目标用户
{用户画像和场景}

## 用户故事

### US-001: {故事标题}
- 作为 {角色}
- 我想要 {功能}
- 以便 {价值}
- **验收标准：**
  - [ ] {标准1}
  - [ ] {标准2}
- **优先级：** {高/中/低}
- **关联需求：** FR-001

### US-002: {故事标题}
...

## 优先级排序
| 编号 | 故事标题 | 优先级 | 状态 |
|------|---------|--------|------|
| US-001 | ... | 高 | 待开发 |

## 非功能性需求
{性能、安全、可用性等要求}
```

### 3. 需求细节确认
如果在生成PRD过程中有不明确的需求细节，直接向用户提问，直到所有细节清晰。

### 4. 写入文档
使用impm_doc_writer将PRD文档写入：
`docs/prds/{项目名称}-prd-v{x.x.x}.md`

### 5. 返回文档位置
返回生成的文档路径给PM agent。

## 交付物
- docs/prds/{项目名称}-prd-v{x.x.x}.md

## 注意事项
- 每个UserStory必须有唯一编号（US-001, US-002...）
- UserStory要与需求文档中的功能需求对应
- 验收标准要具体、可测试
- 所有内容使用简体中文
